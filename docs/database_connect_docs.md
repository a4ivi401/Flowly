# Підключення бекенду до бази даних (MySQL)

## 1. Загальна інформація

Проєкт **AI Time Manager** використовує реляційну базу даних **MySQL** для збереження задач та повʼязаних даних.

- **СУБД:** MySQL (рекомендовано 8.x)
- **База даних:** `ai_time_manager`
- **Основна таблиця (MVP):** `tasks`
- **Роль:** зберігання задач, з якими працює backend та планувальник (Planning Engine).

Цей документ описує, як бекенд має підʼєднуватись до бази даних та які параметри підключення використовувати.

---

## 2. Параметри підключення

### 2.1. Основні параметри

Умовно приймаємо такі значення (адміністратор сервера може задати інші — тоді оновити документ і `.env`):

- **Хост MySQL:**

  - якщо бекенд запущений на тому ж сервері → `127.0.0.1`
  - якщо бекенд на іншому сервері → публічна IP-адреса сервера з MySQL

- **Порт:** `3306` (стандартний порт MySQL)

- **База даних:** `ai_time_manager`

- **Користувач застосунку:** `ai_user`

- **Пароль:** задається адміністратором при створенні користувача, наприклад:
  - `AI_TIME_MANAGER_DB_PASSWORD` (плейсхолдер; реальне значення передається окремо, не в гіті)

---

## 3. Змінні середовища для бекенду

Бекенд читає параметри підключення з `.env` файлу у директорії `backend/`.

### 3.1. Приклад `backend/.env`

```env
# Загальні налаштування застосунку
APP_ENV=dev
APP_HOST=0.0.0.0
APP_PORT=8000

# Параметри підключення до бази даних
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=ai_user
DB_PASSWORD=your_strong_password_here
DB_NAME=ai_time_manager

# Повний URL для SQLAlchemy (FastAPI)
DATABASE_URL=mysql+pymysql://${DB_USER}:${DB_PASSWORD}@${DB_HOST}:${DB_PORT}/${DB_NAME}
```
